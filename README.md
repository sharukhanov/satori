# Сатори — сайт-визитка салона красоты

Статический сайт-визитка салона красоты «Сатори» для размещения на **GitHub Pages**.
Домен: **satori.beauty**.

Сайт — одностраничный лендинг на чистом **HTML + CSS + минимуме ванильного
JavaScript**. Никакой серверной части, базы данных, сборки и админки: владелец
меняет тексты и фото вручную прямо в файлах через веб-интерфейс GitHub.

## Структура репозитория

```
/
├── index.html      ← весь контент и текст сайта (здесь правят тексты/цены)
├── styles.css      ← оформление (цвета, шрифты, адаптив)
├── app.js          ← минимум JS: меню, появление секций, ленивые карты
├── CNAME           ← домен satori.beauty (для GitHub Pages)
├── .nojekyll       ← отключает обработку Jekyll на GitHub Pages
├── robots.txt      ← разрешает индексацию, ссылается на sitemap
├── sitemap.xml     ← карта сайта для поисковиков
├── README.md       ← эта инструкция
└── images/         ← все фотографии (см. images/README.md)
```

## 1. Публикация на GitHub Pages

1. Создайте **публичный** репозиторий на GitHub и залейте в него все файлы
   (этот набор целиком, включая `.nojekyll` и `CNAME`).
2. Откройте **Settings → Pages**.
3. В разделе **Build and deployment** выберите **Source: Deploy from a branch**,
   ветку (`main`) и папку **`/ (root)`**. Сохраните.
4. Через минуту сайт станет доступен по адресу `https://<username>.github.io/<repo>/`.

## 2. Подключение домена satori.beauty

1. В **Settings → Pages → Custom domain** впишите `satori.beauty` и сохраните.
   (Файл `CNAME` в репозитории уже содержит этот домен.)
2. Когда проверка пройдёт — включите галочку **Enforce HTTPS**.

## 3. Настройка DNS у регистратора домена

Пропишите записи для апекс-домена `satori.beauty`:

**A-записи** (IPv4) → укажите все четыре:
```
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
```

**AAAA-записи** (IPv6) → укажите все четыре:
```
2606:50c0:8000::153
2606:50c0:8001::153
2606:50c0:8002::153
2606:50c0:8003::153
```

**Опционально** для поддомена `www`:
```
CNAME  www  →  <username>.github.io
```

DNS может обновляться до 24 часов. После этого сайт откроется по `https://satori.beauty`.

## 4. Как менять тексты вручную (без программиста)

Весь видимый текст находится в файле **`index.html`**. Каждый редактируемый блок
обёрнут парными комментариями на русском:

```html
<!-- РЕДАКТИРУЕМО: текст «О салоне» -->
<p>...здесь текст...</p>
<!-- /РЕДАКТИРУЕМО -->
```

Чтобы изменить текст:

1. Откройте `index.html` на GitHub и нажмите **карандаш «Edit»** (вверху справа).
2. Найдите нужное место по комментарию `<!-- РЕДАКТИРУЕМО: ... -->`
   (поиск по странице — `Ctrl/Cmd + F`).
3. Меняйте **только текст между тегами** (например, между `<p>` и `</p>`).
4. Нажмите **Commit changes**.

> ⚠️ **Важно:** не удаляйте сами теги (`<p>`, `<h2>`, `<li>`, комментарии и т.п.) —
> меняйте только слова между ними. Цены меняются так же — это обычный текст,
> например `от 1 500 ₽`.

Телефоны, адреса, email и ссылки на соцсети вынесены в начало соответствующих
секций и тоже помечены комментариями. У ссылок меняйте значение `href`:
- телефон: `href="tel:+79991234567"`
- WhatsApp: `href="https://wa.me/79991234567"`
- почта: `href="mailto:hello@satori.beauty"`

## 5. Как заменить фото, надпись «Сатори» и видео-лого

Все изображения лежат в папке **`images/`**. Сайт ссылается на конкретные имена
файлов (`hero.jpg`, `service-hair.jpg` и т.д.). Полная таблица со всеми файлами
(включая **каллиграфию «Сатори»** `satori-wordmark.svg` и **видео-лого**
`satori-logo.*` для первого экрана) — в [`images/README.md`](images/README.md).

Два способа заменить картинку:

- **Проще:** загрузите новое фото в `images/` **с тем же именем** (на GitHub:
  **Add file → Upload files**). Оно автоматически встанет на своё место.
- **Или:** поменяйте имя файла в атрибуте `src` нужного `<img>` в `index.html`
  (рядом с каждым `<img>` есть комментарий, какое фото туда вставить).

Пока реального фото нет, на его месте аккуратная заглушка «Фото скоро появится».

## 6. Как вставить карту и карточку Яндекса (раздел «Контакты»)

В разделе «Контакты» зарезервированы два места — для **мини-карты** и для
**карточки организации**. Сейчас там стоят заглушки. Найдите их в `index.html`
по комментариям `<!-- РЕДАКТИРУЕМО: код карты Яндекса -->` и
`<!-- РЕДАКТИРУЕМО: карточка организации ... -->`.

### Мини-карта (можно сделать без профиля бизнеса)

1. Откройте конструктор карт: **constructor.yandex.ru/maps**.
2. Поставьте метку на адрес салона, настройте размер.
3. Нажмите **«Получить код карты»** и скопируйте `<iframe>...</iframe>`.
4. В `index.html` вставьте его **внутрь** блока `<div class="yandex-map" ...>`
   вместо заглушки (`<div class="yandex-placeholder">...</div>`).

### Карточка организации (требуется профиль в Яндекс Бизнесе)

Карточка с **рейтингом, отзывами, фото и часами** берётся из профиля салона в
**Яндекс Бизнесе** (yandex.ru/sprav). Без профиля её сделать нельзя.

1. Заведите/откройте профиль салона в **Яндекс Бизнесе**.
2. Получите код **виджета компании** для сайта.
3. Вставьте его **внутрь** блока `<div class="yandex-card" ...>` вместо заглушки.

### Ленивая загрузка карт (ускоряет сайт)

Чтобы карта подгружалась только при прокрутке до «Контактов», у её `<iframe>`
замените атрибут `src="..."` на `data-src="..."` (допишите `data-` перед `src`).
Скрипт `app.js` сам загрузит карту, когда секция появится на экране. Если
оставить обычный `src` — карта тоже работает, просто грузится сразу.
